# ts-lib-boilerplate

## Purpose
A TypeScript library boilerplate. All public exports are re-exported from `src/index.ts`.

## Entry Point
`src/index.ts` — all public exports are re-exported from here.

## Commands
- `pnpm build` — compile to `dist/` (ESM + CJS + types)
- `pnpm dev` — watch mode
- `pnpm test` — run Vitest (requires `pnpm build` first — tests run against `dist/`)
- `pnpm typecheck` — tsc without emit
- `pnpm lint` — Biome check
- `pnpm format` — Biome format (write)
- `pnpm knip` — dead code check
- `pnpm run docs` — generate TypeDoc to `docs/`

## Structure
- `src/` — source TypeScript
- `dist/` — compiled output (gitignored, generated by build)
- `test/` — Vitest tests
- `docs/` — generated TypeDoc (gitignored)

## Conventions
- All public types must be explicitly annotated (`isolatedDeclarations`)
- Tests run against compiled `dist/`, not raw `src/` — always build before running tests
- Commits follow Conventional Commits for Release Please automation
- No default exports — named exports only

## Tooling
- Package manager: **pnpm** — use `pnpm` instead of `npm`, `yarn`, or `bun`
- Bundler: **tsdown** (Rolldown-backed, ESM-first)
- Lint + Format: **Biome** — single `biome.json`, replaces ESLint + Prettier
- Tests: **Vitest**
- Dead code: **Knip**
- Release: **Release Please** (Conventional Commits → auto Release PR → npm publish)
- Docs: **TypeDoc** with `typedoc-plugin-markdown`
